Summary

google-drive-ftp-adater-online is an open source project. It is a web application allowing you to connect your ftp client to your Google Drive account. This website offers that web application as an ONLINE FREE SERVICE available publicly on Internet. The only aim of this website is to provide a free service.

Please be sure to understand the following

We use your email for 2 reasons: As a username to login to the website and only if needed, as a way to communicate about any issue concerning the application like changes to this privacy policy or security information. Your email will be never used to send you spam or any commercial data. You can delete completely your account yourself including all your personal data at any time.

The application uses a Google Drive Authorization Token (this is not your google username or password). This is a file containing a String allowing the application to connect to your Google Drive account on your behalf, only when you are connected to the FTP Service. You can revoke this authorization at any time yourself.

The application stores the list of your Google Drive files but not the binary content. It means that if you have a file named "your-picture.jpg" or any other file, the application will save into the application's database the path and the name of that file (e.g. "/your-folder-name/your-picture.jpg"), but it wil never store the picture itself or any binary content. The only purpose for caching this list is for perfomance reasons, in order to provide the functionality of this service. This information will be downloaded to application's database the first time you connect your ftp client to the Service. You will have the option to delete completely this data by sending an email to ftpdrive@andresoviedo.org or by deleting your account.

The FTP user is a user you create yourself in the application to allow FTP clients to connect to your google-drive-ftp-adapter-online account, therefore to your Google Drive account. This FTP user should be configured with a username and password different than your Google Credentials or the google-drive-ftp-adapter-online Credentials. For example, you can crate a user "myftpuser" with password "mysecretpassword". You can delete this ftp user yourself at any time. It is your only reponsability to manage that user. We don't assume any responsability derived from the misuse of the FTP user.

If you register and you don't use the ftp service for a period of 1 year your data will be deleted. An email will be sent to you prior to deletion. You will have the option to respond to that email with a consent to mantain your data for 1 more year.

This Service uses a free tier for the Google Drive API and it could be unavailable at any time.

This Service is free and is mantained with no financial support. The Service could be unavailable at any time.

FTP Protocol

The Service provides 2 methods to connect your FTP client to the Service, either using unencrypted FTP or encrypted FTPS. FTP IS NOT A SECURE PROTOCOL because data is transferred with no encryption, while FTPS IS A SECURE PROTOCOL because data is sent through an encrypted connection. We recommend to use FTPS as it is secure and less probably to intercepted by a MAN-IN-THE-MIDDLE attack.
